# 学术论文评审修改

对学术论文进行多层次系统性审查，输出带标注的修订稿和结构化评审报告。

## 输入

支持 `.pdf`、`.docx`、`.tex`、`.md` 格式的论文文件。用户上传文件后，首先读取全文内容。

## 总体原则

- **不改原意**：只在表达层面优化，不改变作者想传达的学术观点
- **不确定就标注**：遇到你无法确认的事项，标记 `[待确认]`，不强行修改
- **引用验证要诚实**：网络未检索到的引用标注为「未检索到」而非断言虚假
- **保持学术严谨**：所有修改建议都给出理由

---

## 阶段一：论文结构识别

读取论文后，首先生成一份"论文结构地图"，包括：

1. **整体元信息**：标题、作者、语言（中文/英文/混排）、文档格式、总页数/字数
2. **章节结构**：按层级列出所有标题及对应起始位置，检查编号连续性
3. **图表清单**：列出所有图表及其编号、标题、首次引用位置
4. **参考文献概况**：文献数量、格式风格（如 IEEE、APA、GB/T 7714）
5. **公式概况**：含公式的章节及公式数量

将此结构地图呈现给用户，确认无误后进入下一阶段。

---

## 阶段二：逐层检查

按以下六轮依次进行，每轮完成后汇总发现的问题，再进入下一轮。

### 轮一：语言与格式（基础校对）

逐段扫描，检查以下项目：

**笔误与错别字**
- 形近字误用（如中英文混排时的常见错误）
- 拼音输入导致的同音错字
- 英文拼写错误（结合上下文判断术语是否正确）

**标点符号**
- 中英文标点混用（中文段落内出现英文逗号/句号等）
- 引号、括号不成对
- 公式后是否缺标点（行内公式视为句子一部分）

**口语化表达**
- 识别并标记口语化、非学术表达，如"搞"、"弄"、"很多"、"非常"（在学术语境下应替换）
- 不必要的程度副词（"十分"、"极其"等）
- 建议替换为更正式、更精确的学术表达
- 英文论文中 colloquial phrases：如 "a lot of" → "many/significant"、"kind of" → "somewhat/rather"

**学术营销用语识别与去情绪化**
学术论文中不应出现带有宣传、煽动、夸耀色彩的营销性语言。应逐段扫描，识别并将此类表达替换为客观、中立的学术表述。

**常见学术营销特征**
- 过度修饰：使用"革命性"、"划时代"、"颠覆性"、"前所未有的"等夸大词汇
- 情绪煽动：使用"令人震惊"、"难以置信"、"众所周知"、"无人能及"等带有情感色彩的表述
- 自夸型评价：对本研究成果使用"极大地提升了"、"完美地解决了"、"显著优于所有方法"等绝对化赞美
- 贬低型比较：对前人工作使用"落后"、"过时"、"低效笨拙"等主观贬义词汇
- 炒作趋势：脱离学术语境地渲染某技术/领域的"火爆"、"井喷"、"爆发式增长"
- 预设结论：在论证尚未完成时就宣称"毫无疑问"、"事实证明"、"显然"等

**识别方法**
- 扫描带有强烈感情色彩的形容词和副词
- 关注"最"类绝对化表述（最好、最高、最大、最优、首次、唯一等），除非有客观数据支撑
- 识别含有价值判断倾向的动词（彰显、印证、奠定了、开辟了）
- 检测结论前置的句式（"毋庸置疑……"、"明显地……"、"众所周知……"）

**去情绪化改写原则**
- 将夸大形容词替换为可量化描述或删除：如"显著的改进" → "准确率提升 4.7 个百分点"
- 将自夸评价改为与具体数据的比较：如"完美解决了" → "在 X 指标上较基线方法 Y 提升了 Z"
- 将贬低型比较改为客观差异描述：如"方法 A 效率低下" → "方法 A 在数据集 B 上的推理时间为 X ms，本文方法为 Y ms"
- 将情绪化预设删除或替换为数据支撑的推断：如"事实证明" → "实验结果表明"
- 将趋势炒作改为带引用的事实陈述：如"该领域近年来飞速发展" → "该领域 2022-2025 年间发表论文数年均增长 23%（来源：……）"
- 英文论文同理：避免 "groundbreaking"、"game-changing"、"state-of-the-art"（除非有 benchmark 榜单支撑）、"to the best of our knowledge" 的过度使用

**改写示例**
| 原文（营销性） | 修改后（客观化） |
|---|---|
| 该方法取得了革命性的突破 | 该方法在 ImageNet 上 top-1 准确率从 84.3% 提升至 87.1% |
| 我们的模型极大地优于所有现有方法 | 我们的模型在 6 个基准数据集中的 5 个上超过了对比方法（见表 3） |
| 实验结果令人震惊地表明 | 实验结果显示 |
| 传统方法完全无法处理此问题 | 传统方法在该场景下的 F1 得分均低于 0.60（见第 4.2 节） |
| 众所周知，深度学习已经改变了世界 | 深度学习在计算机视觉、自然语言处理等领域取得了广泛应用 |
| 该技术引领了一个新时代 | 该技术被后续 120 余篇论文引用并作为基线（截至 2025 年） |
| 毫无疑问，本文方法是最优选择 | 综合 4 项指标，本文方法在 3 项上排名第一（见表 5） |
| 简单粗暴却有效的方案 | 结构简单但性能可比的方案 |

**严重程度判定**
- 全文多处系统性学术营销（如引言、摘要、结论中重复出现）→ 必须修改
- 单次或少量营销用语 → 建议修改
- 仅有轻微倾向、不影响核心传达 → 可选优化

**标题编号**
- 检查编号层级是否正确（如 1 → 1.1 → 1.1.1，无跳号）
- 标题与目录一致性
- 标题风格统一（同级标题的语法结构是否一致）

**图表编号与交叉引用**
- 图表编号是否连续
- 正文中的"如图 X 所示"与实际图号是否对应
- 是否存在只被引用一次或从未被引用的图表

### 轮二：逻辑与结构（中层评审）

**段内逻辑**
- 每段是否有明确的主题句
- 句子间的因果关系、递进关系、转折关系是否清晰
- 是否存在逻辑跳跃（前一句与后一句之间缺少过渡或解释）

**段间衔接**
- 段落之间是否有过渡语句或逻辑承接
- 是否存在"突兀"的段落切换（前一段还在讲方法，下一段突然跳到结论）

**章节内部自洽**
- 每个章节是否完成了标题承诺的内容
- 章节内的论点是否有论据支撑
- 是否存在"说而不证"的关键论断

**全局论证骨架**
- 论文的核心论点是什么？能否从各章节中清晰追踪支撑核心论点的逻辑线？
- 摘要的结论是否与正文结论一致
- 是否有章节游离于核心主线之外

**章节比例**
- 引言/方法/实验/结论各部分篇幅比例是否合理
- 是否有某章过长（细节堆砌）或某章过短（论证不足）

### 轮三：深度学术评审

**术语一致性**
- 同一概念在全文中是否使用相同术语
- 缩写是否在首次出现时给出全称与定义
- 是否存在"一词多义"的歧义（同一术语在不同章节含义不同）

**数学公式准确性检查**
- 公式中每个符号是否在正文中有定义（首次出现时）
- 符号上标、下标、字体（正体/斜体/粗体）是否符合学科惯例
- 公式之间是否存在推导断裂（中间步骤缺失）
- 检查量纲一致性（如有物理量）
- 跨行公式的括号匹配
- 编号公式的引用是否正确

**方法可复现性**
- 实验/方法章节是否给出足够信息让他人复现
- 数据集：来源、规模、划分方式、预处理步骤
- 模型：架构、关键超参数、训练配置
- 评估：指标定义、统计检验方法、显著性水平
- 标出缺少的关键信息项

**实验与结论**
- 实验结果是否支持文中结论
- 是否有选择性报告（只报好结果）
- 对比基线是否公平、有代表性
- ablation study 是否完整

### 轮四：图表质量

逐图检查：
- 清晰度：分辨率是否足够，文字是否可读
- 标注完整性：坐标轴标签、图例、单位
- 传达力：图表是否为传达信息的最优形式（表格 vs. 柱状图 vs. 折线图）
- 自明性：脱离正文是否能大致理解图表含义

### 轮五：数据合理性验证

对论文中引用的数据进行逻辑合理性和准确性检查。必要时通过 WebSearch 访问官方或客观数据源进行交叉验证。

**数据识别**
- 扫描论文全文，提取所有数值型论断，包括：
  - 统计数字（如"2023 年增长 12%"、"覆盖 80% 的用户"）
  - 排名与比较（如"全球第三大"、"首次突破 X"）
  - 绝对值与量级（如"市场规模达 500 亿"、"超 100 万次下载"）
  - 基准数据（如"SOTA 准确率 94.3%"、"基线方法得分为 X"）
- 同时关注非数值型的事实论断（如"首个提出……的方法"、"唯一通过认证的……"）

**内部一致性检查**
- 同一数据在不同章节中出现时数值是否一致
- 图表中的数据与正文描述是否一致
- 百分比加总是否合理（>100% 或明显不构成整体时需要标记）
- 数量级是否合理（如"全球用户 100 人"、"实验用 60 亿样本"可能为笔误）
- 时间逻辑：前后时间线是否存在矛盾（如"2023 年发布，2022 年开始使用"）

**外部交叉验证**
- 对于关键数据论断，使用 WebSearch 工具访问官方或客观数据源验证：
  - 行业统计数据 → 查国家统计局、行业白皮书、权威机构报告
  - 论文基准数据 → 查原论文（如 PapersWithCode、arXiv）、官方排行榜
  - 产品/技术声明 → 查官方网站、工信部备案、专利数据库
  - 学术引用中的数值 → 追溯原始文献核对
- 搜索策略：提取关键词后用 `数据 + 来源关键词 + 年份` 查询

**结果分类**
- ✅ 已确认：找到权威来源且数值在合理误差内
- ⚠️ 存疑：与权威来源有显著偏差（>5% 或量级差异），标注差异具体数值
- ❌ 明显错误：数据自相矛盾或与多个权威来源严重冲突
- 🔍 未验证：无公开可查数据源，标注「建议作者补充数据来源或说明」

**特别注意事项**
- 经济学/社会学数据：注意名义值与实际值的区别、统计口径差异
- 机器学习 benchmark：注意不同评测协议下的结果差异
- 引用他人论文数据时，优先追溯原始出处而非二手引用
- 对"首次"、"最大"、"唯一"等绝对化表述保持警惕，标注后建议作者核实

### 轮六：参考文献校验

**交叉校验**
- 检查正文中引用的每一篇文献是否都在参考文献列表中
- 检查参考文献列表中的每一篇是否在正文中被引用
- 列出不匹配项

**真实性验证**
- 对于关键引用（提出核心算法/理论的引用、用于对比的文献），使用 WebSearch 工具验证其真实存在性
- 搜索策略：用 `"论文标题" + 作者名` 查询
- 结果分类：
  - ✅ 已确认（找到匹配结果）
  - ⚠️ 未检索到（网络搜索无结果，标注为「未检索到，请核实」）
  - ❌ 信息不一致（找到同标题但作者/期刊/年份对不上，标注具体差异）
- 不要求验证全部文献，优先验证核心引用和看似可疑的引用

## 阶段三：生成修订版

在完成六轮检查后，将原稿与修改建议合并，生成修订版。

**标注格式（在原稿内）**

```
[评审标注 | 严重程度：必须/建议/可选 | 问题类型：笔误/逻辑/公式/引用/...]
原文：...出问题的句子...
建议：...修改后的句子...
理由：...为什么这样改...
```

**修订原则**
- 直接可替换的（笔误、标点、编号）→ 给出替换后文本
- 建议重写的（逻辑混乱、表述不清）→ 给出重写建议，标为 `[建议]`
- 需要作者判断的（术语选择、表述风格）→ 给出选项，标为 `[待确认]`

修订稿按原文顺序排列，方便作者逐处处理。

## 阶段四：生成评审报告（.docx）

使用 **python-docx** 库生成结构化的 Word 评审报告。步骤如下：

### 前置准备

```bash
pip install python-docx --break-system-packages
```

### Python 脚本结构

写一个 Python 脚本生成报告，核心代码如下：

```python
from docx import Document
from docx.shared import Inches, Pt, Cm, RGBColor
from docx.enum.text import WD_ALIGN_PARAGRAPH
from docx.enum.table import WD_TABLE_ALIGNMENT
import datetime

doc = Document()

# --- 页面设置 ---
for section in doc.sections:
    section.top_margin = Cm(2.5)
    section.bottom_margin = Cm(2.5)

# --- 样式定义 ---
style = doc.styles['Normal']
font = style.font
font.name = '宋体'
font.size = Pt(11)
```

### 报告结构与生成

**1. 封面**

```python
# 标题
title = doc.add_paragraph()
title.alignment = WD_ALIGN_PARAGRAPH.CENTER
run = title.add_run('学术论文评审报告')
run.font.size = Pt(22)
run.bold = True

doc.add_paragraph()  # 空行

# 论文标题
p = doc.add_paragraph()
p.alignment = WD_ALIGN_PARAGRAPH.CENTER
run = p.add_run(f'论文：{paper_title}')
run.font.size = Pt(14)

# 日期
p = doc.add_paragraph()
p.alignment = WD_ALIGN_PARAGRAPH.CENTER
p.add_run(f'评审日期：{datetime.date.today()}').font.size = Pt(12)

doc.add_page_break()
```

**2. 评审概览**

```python
doc.add_heading('评审概览', level=1)
doc.add_paragraph(f'整体结论：{overall_summary}')

# 各维度评价
doc.add_heading('各维度评价', level=2)
dimensions = ['语言与格式', '逻辑与结构', '方法论', '图表质量', '数据合理性', '参考文献']
for dim, note in zip(dimensions, dim_notes):
    doc.add_paragraph(f'{dim}：{note}')

# 问题统计
doc.add_heading('问题统计', level=2)
doc.add_paragraph(f'必须修改：{count_critical} 项')
doc.add_paragraph(f'建议修改：{count_suggested} 项')
doc.add_paragraph(f'可选优化：{count_optional} 项')
doc.add_page_break()
```

**3. 问题分类列表**

```python
# 问题条目格式：{ 'severity': '必须/建议/可选', 'type': '笔误/逻辑/...',
#                 'location': '第X章第Y段', 'issue': '问题描述',
#                 'suggestion': '修改建议', 'reason': '理由' }

for category, label in [('必须', '必须修改'), ('建议', '建议修改'), ('可选', '可选优化')]:
    items = [i for i in issues if i['severity'] == category]
    if not items:
        continue
    doc.add_heading(label, level=1)
    for idx, item in enumerate(items, 1):
        doc.add_heading(f'问题 {idx}：{item["type"]}', level=2)
        # 用表格展示问题详情
        table = doc.add_table(rows=4, cols=2, style='Light Grid Accent 1')
        table.alignment = WD_TABLE_ALIGNMENT.CENTER
        
        cells_data = [
            ('位置', item['location']),
            ('问题描述', item['issue']),
            ('修改建议', item['suggestion']),
            ('理由', item['reason']),
        ]
        for row_idx, (label, value) in enumerate(cells_data):
            table.cell(row_idx, 0).text = label
            table.cell(row_idx, 1).text = value
            # 第一列加粗
            for p in table.cell(row_idx, 0).paragraphs:
                for run in p.runs:
                    run.bold = True
        
        doc.add_paragraph()  # 间隔
    doc.add_page_break()
```

**4. 附录**

```python
doc.add_heading('附录一：论文结构地图', level=1)
doc.add_paragraph(f'标题：{paper_title}')
doc.add_paragraph(f'语言：{paper_lang}')
doc.add_paragraph(f'章节数：{chapter_count}')
for ch in chapters:
    doc.add_paragraph(ch, style='List Bullet')
doc.add_paragraph(f'图表数：{figure_count}')
doc.add_paragraph(f'参考文献数：{ref_count}')

doc.add_page_break()

doc.add_heading('附录二：引用校验明细', level=1)
for ref in citation_results:
    status_icon = {'confirmed': '✅', 'not_found': '⚠️', 'mismatch': '❌'}.get(ref['status'], '❓')
    doc.add_paragraph(f"{status_icon} [{ref['cite_key']}] {ref['title']} — {ref['authors']} ({ref['year']})")
    if ref['note']:
        doc.add_paragraph(f"   备注：{ref['note']}")
```

### 保存文件

```python
report_path = '论文评审报告.docx'
doc.save(report_path)
print(f'评审报告已保存至: {report_path}')
```

保存到用户可访问的路径，完成后用 `present_files` 呈现给用户。

## 使用方式

用户提供论文文件即可启动全流程。也可以指定只跑某几轮：

- 「只做语言校对」→ 只跑轮一
- 「帮我过一下逻辑」→ 只跑轮二
- 「检查引用」→ 只跑轮六
- 「完整评审」→ 全部六轮 + 报告
